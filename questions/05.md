Subject: Streaming service

Hi,
We have been trying to find some solution to our problem but could not in your documentation. Basically, on our music streaming services, we want logged in users to be able to search their playlist. But to search only their playlist.

We have a "playlist" index that looks like this:

[{
  "name": "Super playlist",
  "user_id": 234
}, {
  "name": "NYE playlist",
  "user_id": 345
}]

Right now we are able to use the "filter" option of Algolia but it's not super secure since any user can change the parameter and access every other user playlist. What can we do to ensure users are only able to search their own playlist and not all of them?

Thanks.

## Your Response

Hi Stan,

You're right, client-side filters alone aren't secure for this. A user who knows their way around dev tools could modify the request and see other people's playlists. The feature you want is **Secured API Keys**, which is designed exactly for this.

The idea is that instead of passing a filter on the frontend (where it can be tampered with), you bake the filter into the API key itself on your backend. That way the restriction is enforced by Algolia no matter what the client sends.

Here's the setup:

**1. Make user_id filterable**

Add **user_id** to **attributesForFaceting** with the **filterOnly** modifier (this lets you filter on it without computing facet counts):

**2. Generate a Secured API Key on your backend**

When a user logs in, create a secured key on your server that locks results to their **user_id**. This filter gets embedded in the key and can't be changed from the frontend:

```js
// On your backend, when user 234 logs in:
const securedApiKey = client.generateSecuredApiKey({
  parentApiKey: 'YourSearchOnlyAPIKey',
  restrictions: {
    filters: 'user_id:234'
  }
});

// Send this key to the frontend for that user's session
```

**3. Use the Secured API Key on the frontend**

On the client side, initialize the search client with the secured key instead of your regular search key:

```js
const searchClient = algoliasearch('YourAppID', securedApiKey);
```

That's it. Now even if a user modifies the search request on their end, the filter is always enforced by Algolia. They can only see their own playlists.

Let me know if you run into anything while implementing this.
