## Your Response

Hi Carrie,

I see what's going on. The issue is that **separatorsToIndex** is an index setting, not a search parameter. Unlike **typoTolerance** and **advancedSyntax**, **separatorsToIndex** can only be configured at the index level through the dashboard or API. It cannot be passed as a search param. The API does not recognize it as a valid query parameter and the whole request fails.

To fix this, remove **separatorsToIndex** from your Configure component in App.js (line 40) and set it as an index setting instead. 

You can do this through the dashboard:

1. Go to your index in the Algolia dashboard
2. Open **Configuration** > **Relevance Essentials** > **Separators to index**
3. Add the characters you need
4. Save your changes

Let me know if that fixes it!
